# 変更履歴（お客様向け）

このファイルは、変更内容をお客様向けに簡潔に記載します。

## v0.3.24 - 2026-06-17
- BOAT RACE ログイン後に「特別なお知らせ」が表示された場合、自動で閉じて開催一覧の操作へ進めるようにしました。
- 「すべて既読にする」は自動では押さず、お知らせ自体は未読のまま残します。

## v0.3.23 - 2026-06-17
- BOAT RACE のログイン画面を開く際、通信が少し遅いだけで2秒タイムアウトしてログイン前に止まる場合がある問題を修正しました。
- 通常の画面操作待ちは従来どおり短く保ち、外部サイトを開く処理だけ長めに待つようにしました。

## v0.3.22 - 2026-06-17
- BOAT RACE の投票TOP画面にログイン済みの状態でも、再ログイン扱いになって発注が止まる場合がある問題を修正しました。
- 開催一覧画面のURLをログイン済みTOPとして認識し、次の発注ではそのまま開催場選択へ進むようにしました。

## v0.3.21 - 2026-06-10
- BOAT RACE の Discord 最終通知に書かれた投資額を、発注準備後もそのまま使うようにしました。
- stable3f の 3連複通知で、受信後に自動投票側の目標払戻設定で金額が再計算されるケースを防ぎました。

## v0.3.20 - 2026-05-12
- 公式サイトへアクセスする際に HTTPS 証明書エラーが発生した場合、ブラウザ接続を作り直して再試行するようにしました。
- 候補連動の事前ログインで `ERR_CERT_AUTHORITY_INVALID` が出た場合でも、処理がそこで止まらないようにしました。

## v0.3.19 - 2026-05-11
- BOAT RACE 公式サイトのサービス時間外にログイン画面を開き続けないよう、中央投票ブラウザの起動時間を `10:00-21:00` に調整しました。
- 起動時間外は、起動済みブラウザを自動で閉じ、発注通知が来てもブラウザを新規起動しないようにしました。
- これにより、サービス時間外表示のままブラウザが残る状態を避け、未使用時間帯のメモリ使用も抑えます。

## v0.3.18 - 2026-05-03
- BOAT RACE の最終通知に `ev=...` や `上位確率 ...` が付いている場合でも、買い目、金額、オッズを読み取れることを確認しました。
- 最終通知の締切時刻からモーニング/日中/ミッドナイト枠を補完し、受信枠を絞っている設定でも該当時間帯の通知を受け取れるようにしました。

## v0.3.17 - 2026-05-03
- BOAT RACE の三連複通知を受信し、公式投票サイトへ `3連複` として入力できることを確認しました。
- 三連複の組番は順不同の買い目として扱い、入力時は公式投票サイトの三連複コードへ変換します。
- 三連複通知の受信から投票画面への受け渡しまで、自動テストで確認できるようにしました。

## v0.3.16 - 2026-04-10
- BOAT RACE の投票内容確認画面で、Playwright の locator 探索が空振りした場合でも `querySelector` で直接入力欄を拾う経路を追加しました。
- 発注失敗時には、スクリーンショットとHTMLに加えて、入力欄ごとの探索結果や確認フォーム抜粋を `ipat_error_context.json` へ保存するようにしました。
- これにより、入力欄が見つからない原因が「要素なし」「locator空振り」「入力不能」のどれかを実データで追いやすくしました。

## v0.3.15 - 2026-04-10
- BOAT RACE の投票内容確認画面で、購入金額と投票用パスワードの入力方法を強化しました。
- 入力欄の探索を、表示中の要素だけでなく DOM 上の要素や `querySelector` による直接取得まで広げ、確認画面で入力欄を見失いにくくしました。
- あわせて、確認画面のエラー表示を整理し、「入力欄がない場合」と「入力欄はあるが入力できない場合」を切り分けて分かるようにしました。

## v0.3.14 - 2026-04-05
- `browser_channel` が空欄でも、Windows ではインストール済み `Chrome`、次に `Edge` を自動で試すようにしました。
- これにより、Chrome が入っているのに設定空欄のため Playwright Chromium 側へ進んでしまうケースを減らしました。
- Windowsの既定ブラウザ設定には依存せず、インストール有無で判定します。

## v0.3.13 - 2026-04-05
- ブラウザ同梱版EXEに、Playwright が `headless` 起動時に使う `chromium_headless_shell` も含めるようにしました。
- これにより、`chrome-headless-shell.exe` が見つからず起動できないケースを抑えました。
- あわせて、同梱版で headless shell が欠けている場合でも、同梱 Chromium 実行ファイルへ自動で切り替えるようにしました。

## v0.3.12 - 2026-04-05
- 一部Windows環境で `%TEMP%` 配下の作成に失敗していたため、Playwright の一時フォルダを自動補正するようにしました。
- `TMP / TEMP / TMPDIR` が使えない場合は、書き込み可能な代替フォルダへ切り替えてからブラウザを起動するようにしました。
- Playwright の実行を専用スレッドへ固定し、`asyncio` ループ上で同期APIが混ざって止まるケースを抑えました。

## v0.3.11 - 2026-04-04
- Windowsの既定ブラウザ起動を Chrome 前提から外し、標準では Playwright Chromium を使うようにしました。
- `browser_channel` や実行ファイルパスを指定している場合でも、起動できなければ Playwright Chromium へ自動で切り替えるようにしました。

## v0.3.10 - 2026-04-04
